## Why no fix was applied

This is a physical layer (Layer 1) issue. The kernel NO-CARRIER flag means the Ethernet PHY detects no link partner. Possible causes:
1. Ethernet cable disconnected from Pi or PLC
2. PLC powered off
3. Ethernet cable damaged
4. PLC Ethernet port failure

No software action (restart, NetworkManager restart, etc.) can restore a physical link. The plc_sensor module will auto-reconnect when the link comes back — no restart needed.

## Action required (human intervention)

1. Check that the Ethernet cable is firmly plugged into both the Pi's eth0 port and the PLC's Ethernet port
2. Verify PLC power — PWR and RUN LEDs should be green
3. Try a different Ethernet cable if available
4. Once link is restored, eth0 should auto-negotiate and the plc_sensor module will reconnect automatically

## NOT restarting viam-server

Restarting would not help — eth0 has no physical link. The module is already retrying and will auto-recover when the cable is reconnected. A restart would only cause unnecessary startup transient errors and risk triggering the false-alarm watchdog cycle documented in previous incidents.
