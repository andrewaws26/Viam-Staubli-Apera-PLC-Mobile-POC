## Fix result
Restart at 13:36:12. By 13:36:23 (11 seconds):

- DNS 3/3, STUN 7/7 UDP + 2/2 TCP
- WebRTC connections succeeded immediately

## Pattern: Tailscale DNS instability
This is the 2nd occurrence in 25 minutes. Tailscale's DNS resolver (fd7a:115c:a1e0::53) intermittently returns "server misbehaving". Go's net.Resolver caches this failure state and does not retry with fallback DNS. The system needs either:
1. A fallback DNS in resolv.conf (e.g., 8.8.8.8) — but Tailscale overwrites resolv.conf
2. Tailscale configured to use a more reliable upstream DNS
3. viam-server to handle DNS failures more gracefully (upstream issue)

## Recommendation
If this keeps recurring, consider adding a cron job or watchdog enhancement that detects this specific pattern (internet reachable + viam-server DNS failures) and auto-restarts viam-server after 2 minutes of sustained failures.
