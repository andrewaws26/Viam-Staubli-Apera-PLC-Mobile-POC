## Recommendation
The watchdog should suppress alerts for the first 30–60 seconds after a viam-server restart, or implement a "recently restarted" grace period. This is the second false alarm (after incident-20260320-111001) caused by startup transients. Consider adding:
- A check for viam-server uptime before alerting on error rate
- Filtering out known startup-transient log messages (signaling "not connected", resource dependency race)
