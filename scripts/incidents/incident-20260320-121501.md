## Recurring recommendation (7th time)
This is a textbook case of the watchdog's known limitations. The same improvements continue to be needed:
1. **Check .prog file mtime** — if a .prog file was modified <30s ago, capture is active
2. **Grace period after PLC reconnect** — suppress alerts for 120s after "🟢 PLC reconnected" log entry
3. **Hysteresis** — require 2+ consecutive failed checks before alerting
4. **viam-server uptime check** — if uptime < 5 min, extend grace period
5. **Incident correlation** — if recovering from a known incident, suppress follow-up alerts for the expected recovery window
6. **Check plate drop logs** — if plate drops are being logged, the system is clearly working regardless of .capture file age
