## Analysis
The watchdog alert was triggered by errors from the old viam-server instance (PID 1215) which was in the process of restarting. By the time the watchdog ran at 12:03:38, the new instance (PID 1420) had already:
- Connected to the PLC (12:03:37)
- Initialized capture at 1Hz
- Passed all STUN network checks
- Begun syncing data to cloud

The 18 PLC failures were accumulated across the lifetime of the old process, not a sudden burst. The cloud DeadlineExceeded was a transient timeout during the restart window.

The "resource missing from dependencies" error for plc-monitor is a normal startup race condition — data_manager was configured before the plc-monitor sensor was constructed. Viam's resource manager handles this by reconfiguring data_manager after plc-monitor is ready (seen at 12:03:36).

## What fixed it
viam-server restart self-resolved all issues. The new instance connected to PLC and cloud within 1 second of constructing resources.

## Recurring recommendation
Same as previous incidents — the watchdog should:
1. Detect viam-server restarts and suppress alerts for 60s post-restart
2. Check .prog file freshness, not just completed .capture files
3. Separate error counts by PID to avoid counting errors from a dead process
4. Add hysteresis (2+ consecutive failed checks before alerting)
