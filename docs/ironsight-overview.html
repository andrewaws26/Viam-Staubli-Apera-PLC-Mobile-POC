The Problem We Solve

No Visibility into Truck Operations

Before: Nobody knew what happened during a shift unless the operator remembered to tell someone. Hours ran, faults fired, and temperatures spiked with no record.

Now: Every engine start, idle period, plate placed, and fault code is recorded automatically — second by second, 24/7.

Reactive Maintenance

Before: We fixed things after they broke. Downtime was the first symptom of every problem.

Now: The AI mechanic spots trends like coolant temperature climbing or battery voltage weakening before failures happen, and gives plain-English recommendations.

No Production Metrics

Before: Plate counts were manual estimates scribbled on a clipboard. Nobody could compare trucks or shifts with real numbers.

Now: Exact counts per shift, per truck, per hour — automatically. Shift reports generate with one click and print for the clipboard.

How It Works

Data is collected once per second from both the engine computer and the PLC. It's stored in the cloud automatically — even if the truck loses internet, data buffers locally (up to 50 MB) and syncs when connection returns. The dashboard is a website accessible from any phone, tablet, or computer. No software to install.

What Happens Automatically (No Human Needed)

  • ✓
    Data collected every second, 24/7 — engine parameters and PLC readings captured continuously while the truck has power
  • ✓
    Offline buffering — if internet drops, data saves locally (up to 50 MB) and syncs automatically when connection returns. No data lost.
  • ✓
    CAN watchdog — system detects when the truck engine is off and sleeps the CAN interface to save power and CPU. Wakes up automatically when the engine starts.
  • ✓
    Auto-recovery — everything starts on power-up. No buttons to press, no software to launch. Plug it in and walk away.
  • ✓
    Fleet sync — code updates deploy to all trucks automatically. Changes pushed to git reach every Pi within 10 minutes via cron sync.
  • ✓
    19 diagnostic rules — the system continuously checks for anomalies across 5 categories (plate flipper, encoder, eject, PLC, operations) and flags issues with plain-English explanations.

Reliability & Edge Cases

What if the truck loses internet?
Data buffers locally on the Pi's SD card (up to 50 MB — roughly 14 hours of continuous readings). When connectivity returns, everything syncs to the cloud automatically. No data is lost.
What if power is cut suddenly?
Both Pis auto-start all services on power-up. The filesystem is journaled to prevent corruption from hard shutoffs. The system is designed for the reality that trucks get their batteries disconnected.
What if the engine is off?
The CAN watchdog detects no engine traffic and puts the CAN interface to sleep, reducing CPU usage from 88% to near zero. It probes every minute and wakes up within 10 seconds of the engine starting. A minimal "vehicle off" payload is still reported.
What if a Pi breaks?
The two Pis are independent — if one fails, the other keeps working. A replacement Pi can be set up in about 30 minutes: flash the SD card, plug it in, and it pulls the latest code automatically.
Does it work on passenger vehicles too?
Yes. The system auto-detects J1939 (heavy trucks) vs OBD-II (passenger vehicles) and adapts. Validated on a 2015 Nissan Altima with 6.6 million CAN frames and zero data drops.
